# Rolan Bernardo

### AI Automation Solutions Architect — Infrastructure · Identity · Cloud

> I design and build automation systems that connect enterprise infrastructure, identity platforms, and AI into reliable end-to-end workflows. My background is hands-on IT engineering across networking, cloud, and identity — which means the automation I architect is grounded in how real systems actually behave in production.

---

## What I Do

I bridge three disciplines that most automation work touches but few practitioners cover in depth:

**Infrastructure, Networking & Cloud**
Enterprise network design and deployment (firewalls, switching, HA/failover, multi-ISP load balancing), AWS, Azure, and Google Workspace administration, secure remote access, and Zero Trust network migrations.

**Identity & Microsoft 365 / Azure**
Entra ID (Azure AD) administration at Global Admin scope, Microsoft 365 governance, automated user provisioning and lifecycle management, SSO integration (SAML / OIDC) with Keycloak, and HR-to-directory attribute synchronization.

**Automation & Scripting**
PowerShell automation across Exchange Online and Microsoft Graph, workflow orchestration with n8n, Power Automate, and Azure Logic Apps, serverless functions, webhook relays, and API integration — increasingly orchestrated with LLMs and agentic tooling.

---

## How AI Fits In

The automation landscape is shifting from rule-based pipelines to AI-augmented systems. My focus is making that shift production-safe:

- Integrating LLMs into operational workflows (triage, summarization, drafting, decision support)
- Building tools and connectors around the **Model Context Protocol (MCP)** to give AI systems controlled access to enterprise data and actions
- Designing automation architectures where AI handles judgment and orchestration handles execution
- Internal productivity tooling — dashboards and assistants that surface actionable signals from email, chat, and ticketing systems
